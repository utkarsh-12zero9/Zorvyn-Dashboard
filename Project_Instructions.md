# 💰 Finance Dashboard UI

## 📌 Objective
This project aims to evaluate frontend development skills by building a clean and interactive **finance dashboard interface**.

The focus is on:
- UI design
- Component structuring
- Frontend data handling  

> ⚠️ This is **not required to be production-ready** or backend-dependent.

> 💡 If you have already built a similar dashboard, you may submit that instead. Include a clear explanation of how it meets the requirements along with repository or deployment links.

---

## 🧭 Key Instructions

- No strict structure — design the layout and components intuitively.
- Keep it **simple and clear**.
- Creativity is encouraged — small thoughtful UI/UX improvements are appreciated.

---

## ⚙️ Flexibility

You are free to:
- Use any frontend framework (React, Vue, Angular, or plain JavaScript)
- Use any styling approach (Tailwind, CSS, MUI, etc.)
- Work with static or mock data
- Structure the project as per your preference

---

## 🧩 Scenario

Build a dashboard that helps users track and understand their financial activity.

The interface should allow users to:
- View overall financial summary
- Explore transactions
- Understand spending patterns

---

## 🚀 Core Requirements

### 1. 📊 Dashboard Overview

Create a main dashboard that includes:
- Summary cards:
  - Total Balance
  - Income
  - Expenses
- At least one **time-based visualization** (e.g., balance trend)
- At least one **categorical visualization** (e.g., spending breakdown)

---

### 2. 📄 Transactions Section

Display a list of transactions with:
- Date  
- Amount  
- Category  
- Type (Income / Expense)

Include basic features:
- Filtering  
- Sorting or Search  

---

### 3. 🔐 Basic Role-Based UI

Simulate roles on the frontend:

- **Viewer**
  - Can only view data  

- **Admin**
  - Can add/edit transactions  

💡 Implement role switching using a dropdown or toggle.

---

### 4. 📈 Insights Section

Display simple insights such as:
- Highest spending category  
- Monthly comparison  
- Any meaningful observation from the data  

---

### 5. 🧠 State Management

Handle application state for:
- Transactions data  
- Filters  
- Selected role  

You may use:
- Context API  
- Redux  
- Zustand  
- Or simple state (if well managed)

---

### 6. 🎨 UI & UX Expectations

- Clean and readable design  
- Fully responsive  
- Graceful handling of empty/no-data states  

---

## ✨ Optional Enhancements

(Not required but recommended)

- 🌙 Dark mode  
- 💾 Data persistence (local storage)  
- 🔌 Mock API integration  
- 🎞️ Animations or transitions
